File Management

No spaces / No uppercase: All file names (index.html, introduction.html, contract.html) are lowercase with no spaces — good.

HTML/CSS Validation

The Accumulus widget shows green on the pages I checked, which is a great sign. Worth confirming with a direct W3C validator run, but it looks like you're passing.

Design Standards

Don't underline non-links: Looks clean.
Brand everything: This is the area that needs the most attention. The site title and home page H1 say "Joseph Murrieta's Jumbo Mastodon," but the Introduction and Contract pages switch to "Alberto Murrieta's Ancient Mastodon." Pick one brand name (and one first name presentation) and use it consistently across every page, header, and <title>.
Link to external files: Excellent — separate CSS files per page (introduction.css, contract.css) linked externally rather than embedded.
Clean up images: The introduction photo is appropriately sized and has a caption — well done.

Page Structure Checklist

Header / Main / Footer: All three semantic elements are present and used correctly. Bonus points for aria-label="Primary" on the nav and aria-current="page" on the active link — that's beyond the requirements and great for accessibility.
Header contents: Site/brand H1 is present, but the required "Student Name * Course ID Course Name" line isn't in the header in the standard format. The course ID does appear in the brand line, which is close but not quite the spec.
Main: Page name is properly used as H2 (e.g., "Course Home Page," "Introduction," "Contract") — exactly right. Unique content per page — good.
Footer requirements: This is the main checklist gap — the footer is missing the HTML validation button/link and the CSS validation button/link. The "Designed by…" requirement is partially met by "Page built by Murrieta Software ©" on the introduction/contract pages (nice touch), but the home page footer doesn't have it. Also, the home page footer has fewer links than the introduction/contract footers — the footer should be the same across the whole site (Repetition principle).

Design Requirements (CRAP, Contrast, Fonts)

Sufficient contrast: Good — the dark text on cream background and the gold/green palette read comfortably.
CRAP principles: Strong on the Introduction and Contract pages (consistent palette, good alignment, nice typography). The home page, however, is essentially unstyled (default browser look) which breaks Repetition badly. Apply your shared site styles to index.html too.
Standard CSS: Good use of external stylesheets and standard properties.

Required Pages

Home (index): Present but very minimal — just an H2 and one sentence. Worth fleshing out and styling to match the rest of the site.
Introduction page: Excellent — has a photo with caption, plus H3 sections for Personal Background, Professional Background, Academic Background, Background in This Course, Primary Computer Platform, Courses I am Taking, a Funny/Interesting item, and a Quote. This hits all the introduction requirements.
Contract page: Present and properly worded — but says "Fall 2026, ITIS 3135" while the course is actually Spring 2026. Quick fix.
Topic/Client Project pages: I noticed footer links to "Hobby" on some pages but couldn't find a corresponding Hobby/topic site in the navigation. If a topic site is required and it exists somewhere, it should be linked from the main nav.

Coding Standards

Indentation / semantic HTML / accessibility: Very clean — proper use of <ul> for nav, ARIA attributes, alt text on the personal photo. This is the strongest part of the submission.
Comments: Couldn't tell from the rendered page; if your sections aren't commented, add some section headers in the source.

The Good / The Bad / The Ugly

The Good: Clean semantic HTML with thoughtful ARIA usage; nicely styled and well-organized Introduction page; good external CSS architecture; readable typography and pleasant color palette on the styled pages; nice personal touches like the tagline ("Never tell me the odds.") and the © branding.
The Bad: Inconsistent name/brand across pages (Joseph vs. Alberto, Jumbo vs. Ancient Mastodon); home page is unstyled and sparse compared to the rest of the site; footer differs across pages (breaks Repetition); contract says Fall 2026 instead of Spring 2026.
The Ugly: Missing HTML and CSS validator links/buttons in the footer — this is an explicit checklist requirement on every page. Also, no clearly linked topic/hobby site from the main nav.

Stop / Start / Continue

Stop

  • Using two different brand names and first names across pages
  • Shipping the contract page with the wrong semester ("Fall 2026")

Start

  • Adding HTML and CSS validator buttons/links to every page footer
  • Applying your shared styles to the home page so it matches the rest of the site
  • Standardizing the footer across all pages
  • Linking the hobby/topic site from the main nav

Continue

  • Using semantic HTML5 elements with bonus ARIA attributes
  • Linking external CSS files per page
  • The warm, personal tone of the Introduction page
  • The clean, readable color palette

Overall Assessment

When the styling shows up (Introduction and Contract pages), this is one of the more polished sites in the class — you clearly know your HTML/CSS. The biggest wins from here are quick: settle on one brand name everywhere, push the same styling to the home page, add the validator buttons to the footer, and standardize the footer across all pages. Great work — keep it up!
